Example 11: repeated fading

To see the difference I will show an example of coding this the old-fashioned way with only fxq and one version also using the new tic methods.

With fxq only:

function fadeInAndOut() {
	$('.example11')​.animate({opacity:0})​.animate({opacity:1})
		.queue(function(next){
			fadeInAndOut();
			next();
		});
}
fadeInAndOut();

With both fxq and tic:

$('.example11')​.repeat()​.animate({opacity:0})​.animate({opacity:1})​.join();
Look at me fading!

I do like the shorter way.

Example 12: Playing pong

var options = { duration:2000, easing:'linear', queue:'myFX' };
$('.example12 .ball')​.repeat()​.animate({left:350},options)​.animate({left:10},options)​.join('myFX');
$('.example12 .ball')​.wait()​.dequeue('myFX');

var options = { duration:900, easing:'linear' };
$('.example12 span')​.repeat()​.animate({top:160},options)​.animate({top:0},options)​.join();
     

The horizontal animation for the ball uses the fxq named 'myFX'.
We have to start that queue manually with the second line using .dequeue('myFX'). Seems to be a jQuery issue.

The vertical animation uses the default fxq which always starts on their own. It includes all three moving elements at once.

Example 13: scheduling different elements' animations

var $items = $('.example13 span');

var X = $$()​.mod($items.length);
$items​.repeat(X)​.text(X.$)​.eq(X)​.animate({opacity:0,top:-50})​.join()​.animate({opacity:1,top:0});
A B C D E

In this example a deferred variable X is used as cycle counter. The selector .eq(X) will only select the element with the current cycle counter's value. Thus the animation iterates.

In each iteration step we wait until the element is shift up using .animate({opacity:0,top:-50}).join()â€‹. Then the next animation is queued, which shifts the element down again. Without waiting the loop starts its next iteration, now using the next element. So we have one element going up and another element going down simultaneously.

In the call of .text(X.$) we have to explicitely use the read-only version, X.$, of our deferred variable. So it is not treated as callback function for .text(). This call is just to see that the loop always starts working on the jQuery selection $('.example13 span'). After the invocation of .eq(X) we work on the Xth element only.
